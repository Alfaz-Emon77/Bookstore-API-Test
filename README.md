# Bookstore API Test Automation (Postman)

## Project Overview
This project demonstrates **API testing of a Bookstore application** using **Postman**.  
It validates CRUD operations (Create, Read, Update, Delete) on books and ensures correct responses and status codes for QA purposes.

> ⚠️ **Note:** This project is for educational/demo purposes only.

---

## Tools & Technologies
- **Postman** – API testing tool  
- **JSON** – Request and response format  
- **Git/GitHub** – Version control and portfolio showcase  

---

## Tested API Endpoints
1. **GET /books** – Retrieve list of all books  
2. **GET /books/{id}** – Retrieve a single book by ID  
3. **POST /books** – Add a new book  
4. **PUT /books/{id}** – Update existing book details  
5. **DELETE /books/{id}** – Delete a book
